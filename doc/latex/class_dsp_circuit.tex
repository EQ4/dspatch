\subsection{Detailed Description}
Workspace for adding and routing components. 

Dsp\-Components can be added to a \doxyref{Dsp\-Circuit}{p.}{class_dsp_circuit} and routed to and from other Dsp\-Components. When a \doxyref{Dsp\-Component}{p.}{class_dsp_component} is added to a \doxyref{Dsp\-Circuit}{p.}{class_dsp_circuit}, a unique string I\-D is optional. This name can be used to identify the component within the circuit's component collection. Connect\-Out\-To\-In and Disconnect\-Out\-To\-In provide a means of routing component outputs to other component inputs, while Connect\-In\-To\-In / Disconnect\-In\-To\-In and Connect\-Out\-To\-Out / Disconnect\-Out\-To\-Out route the circuit's I\-O signals to and from it's internal components.

$\ast$\-N.B. Each component input can only accept one wire at a time. When another wire is connected to an input that already has a connected wire, that wire is replaced with the new one. One output, on the other hand, can be distributed to multiple inputs.

For process intensive circuits, multi-\/threaded processing can be enabled via the Set\-Thread\-Count() method. \doxyref{Dsp\-Circuit}{p.}{class_dsp_circuit} allows the user to specify the number of threads in which he/she requires the circuit to process (0 threads\-: multi-\/threading disabled). A circuit's thread count can be adjusted at runtime, allowing the user to increase / decrease the number of threads as required during execution.

\doxyref{Dsp\-Circuit}{p.}{class_dsp_circuit} is derived from \doxyref{Dsp\-Component}{p.}{class_dsp_component} and therefore inherits all \doxyref{Dsp\-Component}{p.}{class_dsp_component} behavior. This means that a \doxyref{Dsp\-Circuit}{p.}{class_dsp_circuit} can be added to, and routed within another \doxyref{Dsp\-Circuit}{p.}{class_dsp_circuit} as a component. This also means a circuit object needs to be Tick()ed and Reset()ed as a component (see \doxyref{Dsp\-Component}{p.}{class_dsp_component}). The \doxyref{Dsp\-Circuit}{p.}{class_dsp_circuit} Process\-\_\-() method simply runs through it's internal array of components and calls each component's Tick() and Reset() methods.
